# 猫町OSIROイベント設定チェックCLI 外部設計書

## 1. 目的

本システムは、猫町倶楽部のOSIRO管理画面に登録されている募集中イベントを自動確認し、イベント設定の不備をSlackへ通知するCLIアプリケーションである。

確認対象は主にイベントチケットの設定であり、チケット名、金額、販売対象者、オンライン参加URL、主催者からのお知らせ、プラン変更チケットの有無をチェックする。

## 2. 利用者

- イベント設定を確認する運用担当者
- OSIRO管理画面にログインできる管理者
- Slack通知を受け取って修正対応する担当者

## 3. 実行方式

### 3.1 手動実行

プロジェクト直下の以下を実行する。

```powershell
.\run-check.bat
```

またはNode.jsコマンドで直接実行する。

```powershell
npm run check
```

### 3.2 定期実行

Windowsタスクスケジューラーで毎日0時に実行する。

登録済みタスク名:

```text
Nekomachi OSIRO Event Check
```

実行内容:

```text
powershell.exe -NoProfile -ExecutionPolicy Bypass -File "C:\asuka-windows\app\イベントページ自動チェック\scripts\run-check.ps1"
```

タスク設定では「タスクを実行するためにスリープを解除する」を有効にする。
PC側の電源設定でスリープ解除タイマーが無効な場合は、Windowsの電源オプションで有効化する必要がある。

### 3.3 初回ログイン

OSIRO管理画面へのログイン状態を保存するため、初回またはログイン切れ時に以下を実行する。

```powershell
npm run auth
```

ログイン後、`storageState.json` にブラウザのログイン状態が保存される。

## 4. 入力

### 4.1 OSIRO管理画面

イベント一覧URL:

```text
https://nekomachi-club.com/admin/events?state=yet_end
https://nekomachi-club.com/admin/events?limit=30&page=2&state=yet_end
```

取得する主な情報:

- イベント名
- 詳細URL
- 開始日時
- 終了日時
- 会場
- チケット名
- チケット金額
- 販売対象者
- オンライン開催有無
- オンライン参加URL
- 主催者からのお知らせ

### 4.2 環境変数

`.env` に以下を設定する。

```env
SLACK_BOT_TOKEN=xoxb-xxxxxxxxxxxxxxxx
SLACK_CHANNEL_ID=C0BCXMXG745
HEADLESS=true
```

### 4.3 ルール設定

`config/rules.yaml` にオンライン・オフラインごとの基本チケットルールを定義する。

販売対象者タグ:

```text
オン: オンライン会員
オフ: 地域会員
ハイ: ハイブリッド会員
外: 非会員
A / U-22 / B: 旧会員
```

## 5. 出力

### 5.1 コンソール出力

チェック結果のサマリーとNG詳細を標準出力に表示する。

### 5.2 Slack通知

Slackに以下を通知する。

- チェック対象件数
- 対象外件数
- OK件数
- NG件数
- 実行日時
- NGイベント一覧
- イベント名と開催日時
- イベント種別
- 詳細URL
- 不備内容

NGイベントは開催日時の降順で表示する。開催日時を取得できないイベントは末尾に表示する。

### 5.3 ログファイル

`run-check.bat` またはタスクスケジューラー経由で実行した場合、`logs/` にログを保存する。

ファイル名:

```text
logs/check-YYYYMMDD-HHMMSS.log
```

### 5.4 障害調査用ファイル

詳細取得に失敗した場合、可能な範囲で以下を保存する。

```text
artifacts/screenshots/
artifacts/html/
```

## 6. チェック対象イベント

OSIRO管理画面の募集中イベント一覧から、詳細URLが以下形式のものを対象にする。

```text
/admin_events/{id}/edit
```

以下の操作リンクはイベント名として扱わない。

- 削除する
- 非公開にする
- 複製する
- 編集する
- 参加者
- 分析
- チケット

## 7. イベント種別判定

### 7.1 対象外

イベント名に以下を含む場合は対象外とする。

- 予告
- 一覧
- 事務局決済

### 7.2 オフライン

イベント名に以下を含む場合はオフラインイベントとする。

- 【東京】
- 【大阪】
- 【京都】
- 【福岡】
- 【福岡 第一回】
- 【名古屋】
- 東京開催

`〖東京〗` のような括弧表記は `【東京】` と同等に扱う。

### 7.3 オフ＋オン

会場が以下の場合は、オフ＋オン開催イベントとする。

```text
オフ会場＋オンライン
```

オフ＋オン開催イベントでは、オフラインイベントのチェックとオンラインイベントのチェックを両方実施する。

### 7.4 オンライン

対象外・オフライン・オフ＋オンに該当しないイベントはオンラインイベントとする。

## 8. イベント種別ごとのチェック内容

### 8.1 チェック適用サマリー

イベント種別ごとに実施する通常チェックは以下のとおりである。

| イベント種別 | 基本チケット | オンライン参加URL・主催者からのお知らせ | オフライン参加種別 | 備考 |
| --- | --- | --- | --- | --- |
| オンライン | オンライン基本チケット | 実施する | 実施しない | 対象外・オフライン・オフ＋オンに該当しないイベント |
| オフライン | オフライン基本チケット | 実施しない | 実施する | イベント名に地域名や東京開催を含むイベント |
| オフ＋オン | オフライン基本チケット | 実施する | 実施する | 会場が `オフ会場＋オンライン` のイベント |

例外チケットや例外イベントによる対象外条件は、9章の共通チェックで定義する。

### 8.2 オンライン基本チケット

オンラインイベントでは、イベント内に以下のチケットが存在することを確認する。

| 単位 | 対象 | チェック内容 |
| --- | --- | --- |
| イベント単位 | 基本チケット | 以下の会員種別・条件のチケットが存在すること |
| チケット単位 | 各チケット | 金額、販売対象者、チケット名が期待値と一致すること |

| 会員種別 | 条件 | 金額 |
| --- | --- | ---: |
| オンライン会員 | 1回目 | 0円 |
| オンライン会員 | 2回目以降 | 800円 |
| 地域会員 | 通常 | 800円 |
| ハイブリッド会員 | 通常 | 0円 |
| 非会員 | 通常・初参加 | 1,100円 |

非会員チケットは、通常チケットと初参加チケットの2つが存在する場合は許容する。

### 8.3 オンライン参加URL・主催者からのお知らせ

| 単位 | 対象 | チェック内容 |
| --- | --- | --- |
| チケット単位 | プラン変更チケット以外でオンライン開催ONのチケット | オンライン参加URLが入力されていること |
| イベント単位 | プラン変更チケット以外でオンライン開催ONのチケット間 | オンライン参加URLが一致していること |
| イベント単位 | プラン変更チケット以外のチケット間 | 主催者からのお知らせが一致していること |
| チケット単位 | 主催者からのお知らせ | 締切時刻がイベント開始5分前であること |

締切時刻の例:

```text
18:30開始 -> 18:25までに
```

### 8.4 オフライン基本チケット

オフラインイベントでは、イベント内に以下のチケットが存在することを確認する。

| 単位 | 対象 | チェック内容 |
| --- | --- | --- |
| イベント単位 | 基本チケット | 以下の会員種別・条件のチケットが存在すること |
| チケット単位 | 各チケット | 金額、販売対象者、チケット名が期待値と一致すること |

| 会員種別 | 条件 | 金額 |
| --- | --- | ---: |
| 地域会員 | 1回目 | 0円 |
| ハイブリッド会員 | 1回目 | 0円 |
| 地域会員 | 2回目以降 | 1,800円 |
| ハイブリッド会員 | 2回目以降 | 1,800円 |
| オンライン会員 | 通常 | 1,800円 |
| 非会員 | 通常・初参加 | 2,300円 |

非会員チケットは、通常チケットに加えて初参加チケットが存在する場合も許容する。

### 8.5 オフライン参加種別

オフライン読書会では、参加種別として `読書会のみ参加` と `懇親会まで参加` を確認する。

| 単位 | 対象 | チェック内容 |
| --- | --- | --- |
| イベント単位 | イベント全体 | `読書会のみ参加` と `懇親会まで参加` が存在すること |
| イベント単位 | 各会員種別 | `読書会のみ参加` と `懇親会まで参加` が1つずつ存在すること |
| イベント単位 | 非会員 | `通常` には `読書会のみ参加` と `懇親会まで参加` が1つずつ存在すること。`初参加` チケットが存在する場合も同じ2種類が1つずつ存在すること |
| イベント単位 | 重複判定 | 同じ会員種別で同じ参加種別が複数ある場合はNG。参加種別違いの2チケットのみ許容する |

### 8.6 ゲストイベント

イベント名またはチケット構成からゲストイベントと判断できる場合は、通常料金ではなくゲストイベント料金で確認する。

| 単位 | 対象 | チェック内容 |
| --- | --- | --- |
| イベント単位 | ゲストイベント判定 | イベント名またはチケット構成からゲストイベントか判定する |
| チケット単位 | オンラインゲストイベントの各チケット | オンラインゲスト料金に一致すること |
| チケット単位 | オフラインゲストイベントの各チケット | オフラインゲスト料金の金額セットのいずれかに一致すること |

オンラインゲスト料金:

| 会員種別 | 条件 | 金額 |
| --- | --- | ---: |
| オンライン会員 | 1回目 | 550円 |
| 地域会員 | 通常 | 1,200円 |
| オンライン会員 | 2回目以降 | 1,200円 |
| ハイブリッド会員 | 通常 | 550円 |
| 非会員 | 通常・初参加 | 1,500円 |

チケット名として `地域会員・オンライン会員（今月2回目以降）` のように1枚にまとまっている場合、そのチケットは `地域会員 通常` と `オンライン会員 2回目以降` の両方に該当するものとして扱う。地域会員自体には1回目・2回目以降の条件は付けない。

オフラインゲスト料金:

| 会員種別 | 条件 | 金額セット1 | 金額セット2 |
| --- | --- | ---: | ---: |
| 地域会員・ハイブリッド会員 | 1回目 | 800円 | 500円 |
| 地域会員・ハイブリッド会員 | 2回目以降 | 3,000円 | 2,300円 |
| オンライン会員 | 通常 | 3,000円 | 2,300円 |
| 非会員 | 通常・初参加 | 3,500円 | 2,800円 |

## 9. 共通チェック

### 9.1 回数表記

| 単位 | 対象 | チェック内容 |
| --- | --- | --- |
| チケット単位 | 1回目チケット | チケット名に `今月1回目` が含まれていること |
| チケット単位 | 2回目以降チケット | チケット名に `今月2回目以降` が含まれていること |

このチェックはオンライン、オフライン、ゲストイベントのいずれでも、会員種別ルールに `1回目` または `2回目以降` が設定されているチケットに対して実施する。

### 9.2 申込締切

| 単位 | 対象 | チェック内容 |
| --- | --- | --- |
| イベント単位 | 申込締切 | `申込締切：` の日付が、開催日の3日前から開催日までの範囲内であること |

`締切を設定する` がOFFのイベントは、このチェックの対象外とする。

例:

```text
開催日: 2026/07/22 -> 申込締切日: 2026/07/19〜2026/07/22
```

### 9.3 プラン変更チケット

複数チケットのイベントでは、旧会員向けのプラン変更チケットが必要である。

| 単位 | 対象 | チェック内容 |
| --- | --- | --- |
| イベント単位 | イベント全体 | プラン変更チケットが存在すること |
| チケット単位 | プラン変更チケット | チケット名が指定文言に一致すること |
| チケット単位 | プラン変更チケット | 販売対象者に `A`、`U-22`、`B` が含まれていること |
| イベント単位 | イベント内の全チケット | `A`、`U-22`、`B` がプラン変更チケット以外に含まれていないこと |

プラン変更チケット名:

```text
プラン変更後にお申し込み下さい。プラン変更前は参加ボタンは押さないでください。
```

### 9.4 1チケットのみのイベント

| 単位 | 対象 | チェック内容 |
| --- | --- | --- |
| イベント単位 | チケットが1つだけのイベント | チケットが無料であること |

この場合、プラン変更チケット、主催者からのお知らせ、オンラインURLなどの通常チェックは行わない。

ただし、チケット名に `お申し込み済みの方` を含む特殊申込済みチケットの場合は、無料でなくてもOKとし、金額チェックは行わない。

### 9.5 固定費チケット

| 単位 | 対象 | チェック内容 |
| --- | --- | --- |
| イベント単位 | チケットが2つのイベント | 片方がプラン変更チケットであること |
| チケット単位 | 固定費チケット | 金額チェックは行わない |
| チケット単位 | 固定費チケット | 販売対象者に `オン`、`オフ`、`ハイ`、`外` が含まれていること |

チケットが2つで片方がプラン変更チケットでない場合はNGとする。

### 9.6 特殊申込済みチケット

チケット名に `お申し込み済みの方` を含むチケットを、特殊申込済みチケットとして扱う。

| 単位 | 対象 | チェック内容 |
| --- | --- | --- |
| チケット単位 | 一部のチケットだけが特殊申込済みチケットの場合 | 重複チェックと金額チェックから除外する |
| チケット単位 | 一部のチケットだけが特殊申込済みチケットの場合 | 販売対象者、オンラインURL、主催者からのお知らせは通常どおり確認する |
| イベント単位 | 複数チケットかつ全チケットが特殊申込済みのイベント | `9.5 固定費チケット` と同じく例外イベントとして扱い、通常の会員プラン別チェックは行わない |
| イベント単位 | 全チケットが特殊申込済みのイベント | 金額チェック、販売対象者、オンラインURL、主催者からのお知らせの通常チェックは行わない |
| イベント単位 | 1チケットのみの特殊申込済みイベント | `9.4 1チケットのみのイベント` を優先し、無料チェックも対象外とする |

イベント単位の条件に該当する場合は、チケット単位の通常チェックよりもイベント単位の扱いを優先する。
全チケットが特殊申込済みの場合、固定費チケットとしてのプラン変更チケット確認や販売対象者確認も行わない。

### 9.7 全N回チケット

| 単位 | 対象 | チェック内容 |
| --- | --- | --- |
| チケット単位 | チケット名に `全N回` を含むチケット | 金額チェックから除外する |
| イベント単位 | プラン変更チケットを除く全チケットが `全N回` チケットのオンラインイベント | オンライン基本チケットではなく、オンライン会員、地域会員、ハイブリッド会員、非会員のチケットが1つずつ存在すること |

`N` は数字とする。例: `全6回`

プラン変更チケットを除く全チケットが `全N回` チケットの場合、通常のオンライン基本チケット、1回目・2回目以降の構成チェックは行わない。
プラン変更チケットが存在する場合は、プラン変更チケット自体を通常どおり確認する。存在しない場合でも、全N回イベントではプラン変更チケットを必須とはしない。
1枚のチケットに複数の販売対象者タグがある場合は、複数の会員種別を満たすものとして扱う。
ただし、同じ会員種別が複数チケットに含まれる場合はNGとする。非会員は通常チケットと初参加チケットの2つが存在する場合のみ許容する。

### 9.8 書名チェック

イベント名から抽出した書名と、チケット名に含まれる書名を比較する。

| 単位 | 対象 | チェック内容 |
| --- | --- | --- |
| チケット単位 | チケット名 | イベント名と異なる本のタイトルが含まれている場合はNG |
| チケット単位 | チケット名 | `読書会なし` と `読書会セット` は本のタイトルとして扱わない |

対象とする括弧:

- 『』
- 「」
- 《》
- 〈〉

### 9.9 運営メンバーチケット

運営メンバーチケットは、通常の会員向けチケットではなく、初心者読書会・ビギナー限定イベントで追加で必要になる例外チケットである。

#### イベント単位のチェック

| 対象 | チェック内容 |
| --- | --- |
| イベント名またはチケット名に `初心者読書会` または `ビギナー限定` を含むイベント | `運営メンバー` チケットが存在すること |

#### チケット単位のチェック

| 対象 | チェック内容 |
| --- | --- |
| `運営メンバー` チケット | 金額が0円であること |

#### 8章との関係

| 関係する章 | 扱い |
| --- | --- |
| `8.2 オンライン基本チケット` | 通常の会員プランチェックには含めない |
| `8.3 オンライン参加URL・主催者からのお知らせ` | オンライン開催ONの場合はチェック対象に含める |
| `8.4 オフライン基本チケット` | 通常の会員プランチェックには含めない |
| `8.5 オフライン参加種別` | オフライン参加種別チェックには含めない |

## 10. エラー表示

チケット単位のエラーには、上から何番目のチケットかを表示する。

例:

```text
[3番目] チケット「地域会員」: 金額が期待値と異なります。期待: 800円 / 実際: 1100
```

主催者からのお知らせが1チケットだけ異なる場合は、該当チケット名と位置を表示する。

## 11. 運用上の注意

- Slack投稿先チャンネルにはBotを招待しておく。
- `storageState.json` はログイン情報を含むためGit管理しない。
- OSIRO管理画面のHTML構造が変わると、詳細取得に失敗する可能性がある。
- ログイン期限が切れた場合は `npm run auth` を再実行する。
